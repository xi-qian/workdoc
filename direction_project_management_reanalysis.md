# 基于公司方向与项目管理任务分解的运营再分析

## 一、分析依据

本分析在前序审批数据、周报数据基础上，进一步结合两个文件：

- `../材料总结.pdf`：公司方向阐述，核心包括“平台型管线公司”“买种-育苗-卖苗”“AI for Value / Science / Organisation / Market”。
- `../项目管理任务分解.xlsx`：项目全生命周期任务规划，覆盖项目立项、概念验证、工艺研究、IIT 临床阶段、收费阶段、IND、临床试验、上市等阶段。

本分析重点回答三个问题：

1. 按公司方向看，当前运营问题的本质是什么？
2. 按项目管理任务分解看，应该补哪些管理动作？
3. 应该设置哪些指标追踪执行效果？

## 二、公司方向的核心理解

### 1. 公司不是单纯项目制服务公司

从 `材料总结.pdf` 看，公司方向更接近“平台型管线公司”：

- 以种子管线获取为入口。
- 通过概念验证形成证据型资产。
- 通过阶段性 BD、License-out、注册路径和产业合作实现价值回收。
- 核心不是单个医院项目完成，而是持续沉淀可复用、可交易、可注册转化的管线资产。

### 2. 价值孵化模型是“买种-育苗-卖苗”

可以理解为：

- **买种**：获取早期项目和临床需求入口，包括医院、KOL、科研合作、AI/虚拟细胞筛选等。
- **育苗**：通过概念验证、体内外实验、临床试点、CMC、Potency、安全性、数据沉淀，把早期机会变成证据型资产。
- **卖苗**：通过阶段性交易、BD、License-out、FDA/CDE 路径、全球合作等实现价值回收。

这意味着，公司管理重点不只是“项目有没有推进”，而是每个项目是否在持续增加证据资产和交易价值。

### 3. AI for Value 是总系统

PDF 中把 AI for Value 拆成三类引擎：

- **AI for Science**：从靶点发现、干实验、湿实验、动物实验、人体临床试验到数据回流，形成科学验证闭环。
- **AI for Organisation**：从业务事件到经营指标改善，打通任务、流程、责任、规则、证据和反馈。
- **AI for Market**：从临床需求、医院/KOL反馈、真实世界证据、BD需求、政策支付监管和资本市场表达，形成市场价值闭环。

因此，管理指标也应覆盖三类资产：

- 产品/科学证据资产。
- 组织/流程证据资产。
- 市场/交易证据资产。

## 三、项目管理任务分解的核心理解

`项目管理任务分解.xlsx` 给出了公司项目全生命周期的阶段结构：

1. **项目立项**
   - BD 开发：合作意向。
   - 需求确认：需求会议纪要。
   - 内部立项评估：项目可研报告、项目计划、预算、周期等。

2. **概念验证**
   - 实验方案与设计。
   - 体内、外实验验证。
   - 动物安全性评价。

3. **工艺研究**
   - 工艺设计方案。
   - 工艺小试。
   - 工艺放大。
   - 质检方法和质量标准。

4. **IIT 临床阶段**
   - 材料准备：研究方案、资质文件、医院模板类文件。
   - 医院审批：科学评估、伦理审批、签订合同。
   - 备案：准备备案材料及备案。
   - 受试物准备：工艺研究、细胞生产与质控。
   - 临床运营：中心启动、首例入组、完成入组、关闭临床。
   - 数据统计：统计方法、数据文件、知识产权确认。
   - 项目关闭：资料归档与关闭。

5. **收费阶段**
   - 国家审批。
   - 合作协议。
   - 收益分配。
   - 放大生产。
   - 收费治疗。
   - 归档。

6. **IND 及后续**
   - 非临床补充研究。
   - 产品生产和质量研究。
   - 资料汇总。
   - IND 批件。
   - 临床试验。
   - 上市。

这个任务分解不是要求所有工作严格串行推进。实际执行中，很多项目会出现多个阶段任务并行：一边做医院沟通和伦理材料，一边推进合同、CRO方案、预算测算、工艺准备或潜在BD判断。因此，更准确的管理模型应是“阶段门控 + 并行任务流”：允许并行，但每条任务流必须有进入条件、依赖关系、风险标记、责任人和停止点。

## 四、重新定义后的当前主要问题

### 问题一：潜在新业务很多，但“买种”标准不足

#### 现象

周报中大量潜在新业务被临时放在 `IDO项目` 下，原因是还没有正式项目号。例如：

- 东升方向需要沟通商业模式，并出具核心阶段、周期、成本供测算。
- 天津肿瘤医院 GCP 主任方向仍在挑选靶点。
- 广州祈福医院方向仍在基于前期数据挑选靶点和适应症人群。
- 南京鼓楼、复旦中山等 mRNA 增强 NK/CAR-NK 方向存在无合适方案或需外部研发合作的问题。

#### 按公司方向看

这些都属于“买种”阶段。但当前问题是，种子机会进入管线池前，缺少统一的准入判断。

#### 主要缺口

- 是否有明确临床未满足需求。
- 是否有可行靶点、细胞类型和技术路径。
- 是否有可验证的概念验证方案。
- 是否有潜在 BD、License-out 或注册转化路径。
- 是否有预算、周期和成功概率初评。
- 是否能形成可沉淀的证据资产。

#### 措施

建立“潜在新业务准入卡”，所有未建项目号的潜在新业务先进入机会池，完成准入卡后再决定是否创建正式项目。

准入卡至少包含：

- 临床需求和患者人群。
- 医院/KOL/合作方情况。
- 技术路径和适应症。
- 已有证据基础。
- 概念验证设计。
- 预算、周期和核心资源。
- 主要风险。
- 潜在价值路径：BD / License-out / CDE / FDA / 收费治疗 / 数据资产。

### 问题二：项目立项评估没有充分前置，导致后续反复修改

#### 现象

周报中多次出现方案修改、预算压缩、路径变化和医院流程变化：

- 血研所 MRD 阳性 AML NK 项目需要调整至预算可控版本。
- 江苏省肿瘤医院项目因费用未计算完，暂未提交新方案。
- 血研所 MSC 代谢综合征项目因招募和院内检查费用高，需要重新修改方案。
- 中山一院项目因适应症与上海六院重复，备案存在风险甚至暂停。
- 部分项目因医院只接受已上市药品 IIT、伦理会议周期固定、院内行政推动不足而停滞。

#### 按项目任务分解看

这些问题本应在“项目立项-内部立项评估”阶段处理。Excel 中明确要求输出项目可研报告、项目计划、预算、周期等。

#### 主要缺口

- 可研报告不够标准化。
- 预算和周期测算滞后于医院沟通。
- 医院流程和备案路径没有在立项前确认。
- 技术路径和适应症重复风险未充分排查。
- CRO 能力、成本和交付周期未作为立项门槛。

#### 措施

建立正式项目立项 Gate 0。没有通过 Gate 0 的项目，原则上只能停留在潜在新业务池；如因医院窗口期、关键KOL沟通、CRO排期等原因需要先行推进部分任务，必须标记为“先行任务”，明确先行原因、预算上限、风险承担人、补齐 Gate 0 的截止时间和停止条件。

Gate 0 必须输出：

- 项目可研报告。
- 项目计划。
- 阶段预算。
- 里程碑周期。
- 医院流程图。
- 备案路径判断。
- 关键风险和退出条件。

### 问题三：概念验证和证据资产沉淀不足

#### 现象

周报中很多项目处在靶点、适应症、方案、CRO、材料准备阶段，但不容易看出每个项目已经沉淀了哪些可复用证据资产。

例如：

- mRNA 增强 NK/CAR-NK 方向多次出现无合适方案。
- 江苏省肿瘤医院项目学术审查要求补充剂量爬坡、质控设计、不良反应应急预案等。
- 胸科医院项目卡在胎盘知情同意合规性。
- 多个项目依赖 CRO 修改方案，但缺少内部证据包标准。

#### 按公司方向看

“育苗”的核心不是推进动作本身，而是把早期项目培育成可识别的证据型资产。AI for Science 要求靶点发现、实验验证、临床数据、模型迭代和数据回流闭环。

#### 主要缺口

- 项目证据包定义不足。
- 干实验、湿实验、动物实验、临床试点之间的证据链不清。
- CMC、Potency、安全性、临床转化证据没有作为阶段性资产管理。
- 数据回流和复盘机制不足。

#### 措施

为每个正式项目建立“证据资产包”，按阶段沉淀。

证据资产包至少包括：

- 临床需求证据。
- 靶点和机制证据。
- 体内/体外实验数据。
- 动物安全性和有效性数据。
- 工艺和质控数据。
- CMC / Potency / 安全性资料。
- 临床试点数据。
- IP / 论文 / 数据包。
- 市场和 BD 反馈证据。

### 问题四：项目全生命周期“阶段门控 + 并行任务流”没有真正落地

#### 现象

周报按自然语言描述项目进度，但审批数据中反复出现：

- 项目编号或预算归属不清。
- 合同、付款、采购材料滞后。
- 线下合同、后补协议、后补发票、后补回单。
- 条件通过后需要人工跟踪。
- 大额资金事项缺少前置项目和现金流说明。

#### 按项目任务分解看

Excel 已经定义了从项目立项到 IIT、收费阶段、IND 的阶段任务。但当前管理更像“按事件推进”，而不是“按阶段门控和任务流依赖关系管理”。问题不在于多个阶段并行本身，而在于并行任务缺少依赖关系、风险状态和补证机制。

#### 主要缺口

- 每个项目当前所处的主阶段不够清晰。
- 并行推进的任务流、依赖条件和先行风险不够清晰。
- 阶段进入/退出标准不够明确。
- 合同、付款、采购没有嵌入项目里程碑。
- 项目关闭和资料归档没有成为硬性节点。

#### 措施

建立项目“阶段门控 + 并行任务流”台账。

建议主阶段：

- Gate 0：潜在新业务准入。
- Gate 1：项目立项。
- Gate 2：概念验证启动。
- Gate 3：概念验证完成。
- Gate 4：工艺确认完成。
- Gate 5：IIT 材料准备完成。
- Gate 6：医院科学评估/伦理通过。
- Gate 7：备案完成。
- Gate 8：中心启动。
- Gate 9：首例入组。
- Gate 10：完成入组。
- Gate 11：数据统计和 IP 确认。
- Gate 12：项目关闭归档。
- Gate 13：收费/BD/License-out/IND 决策。

每个 Gate 必须有交付物、责任人、预算状态、审批状态和风险状态。同时，允许并行推进的任务需要单独记录：

- 任务流名称：如医院审批、合同、CRO、工艺、采购、预算、BD。
- 所属主阶段。
- 是否先行。
- 先行原因。
- 依赖条件。
- 风险等级。
- 预算上限。
- 责任人。
- 截止时间。
- 停止条件。

### 问题五：AI for Organisation 的治理闭环还没有形成

#### 现象

审批和周报都显示，公司有大量管理事件：

- 采购、合同、付款、预算、临床运营、审批、风险事件、经营财务数据。
- 但这些事件尚未完全转化为统一的经营指标和管理闭环。

审批数据中的典型表现：

- 条件通过事项多，但关闭状态依赖评论。
- 材料补充往返多。
- 线下合同和台账补录多。
- 项目/预算归属需修正。
- 供应商准入状态不一致。

#### 按公司方向看

AI for Organisation 的目标是从业务事件到经营指标改善：任务、流程、责任、规则、证据、反馈形成闭环。

#### 主要缺口

- 审批条件没有统一看板。
- 项目任务和审批任务没有打通。
- 风险事件没有沉淀为规则更新。
- 指标没有反向驱动流程优化。

#### 措施

建立“运营治理看板”，把项目、审批、预算、合同、供应商、风险事件统一管理。

看板至少包括：

- 项目阶段和里程碑。
- 审批条件闭环。
- 材料补充次数。
- 合同和付款风险。
- 预算和项目归属异常。
- 供应商准入异常。
- 项目延误和外部等待。
- 风险复盘和规则更新。

### 问题六：AI for Market 的市场价值证据不足

#### 现象

周报中很多项目有医院/KOL接触、意向、方案讨论和潜在合作，但尚未形成统一的市场价值表达：

- 哪些临床需求最强。
- 哪些医院/KOL能形成示范。
- 哪些项目有 BD 或 License-out 潜力。
- 哪些项目能走收费治疗或注册路径。
- 哪些证据能用于融资、资本市场表达或对标交易。

#### 按公司方向看

AI for Market 不是简单营销，而是把临床需求、产业合作、资本市场表达组织成价值转化闭环。

#### 主要缺口

- 市场需求证据没有结构化。
- KOL/医院反馈没有形成证据包。
- BD 机会、授权机会、融资价值没有量化。
- 试点项目和真实世界证据没有统一沉淀。

#### 措施

建立“市场价值证据包”，与科学证据包并行。

市场价值证据包至少包括：

- 临床未满足需求。
- 医院/KOL反馈。
- 患者场景和入组可行性。
- 真实世界证据。
- 政策、支付和监管环境。
- 对标公司和交易案例。
- 潜在 BD / License-out 对象。
- 价值主张和资本市场表达材料。

## 五、当前应优先采取的措施

### 措施一：把潜在新业务从周报描述变成机会池

所有暂未建项目号的业务机会统一进入“潜在新业务池”，不再只用自然语言散落在周报里。

字段建议：

- 机会名称。
- 来源医院/KOL/合作方。
- 临床需求。
- 技术路径。
- 当前证据。
- 下一步验证动作。
- 预算初估。
- 周期初估。
- 成功概率。
- 潜在价值路径。
- 是否建议创建正式项目。

### 措施二：正式项目必须通过 Gate 0

任何项目进入正式推进前，原则上必须完成 Gate 0 准入评估。

Gate 0 未通过但确需先行推进的事项，应作为“先行任务”管理，不应默认进入常规合同、付款、采购和医院正式提交流程。

### 措施三：建立项目“阶段门控 + 并行任务流”台账

把 Excel 中的全生命周期任务转为可执行台账：

- 每个项目当前主阶段。
- 并行任务流及其依赖关系。
- 当前主阶段和任务流应交付物。
- 实际交付物。
- 责任人。
- 计划完成时间。
- 当前风险。
- 是否允许进入下一阶段。
- 是否存在先行任务及其停止条件。

### 措施四：建立双证据包机制

每个项目都要同时沉淀：

- 科学证据包：靶点、机制、实验、工艺、质控、安全性、临床数据。
- 市场证据包：临床需求、医院反馈、KOL、BD、支付、政策、对标交易、资本表达。

### 措施五：把审批条件纳入项目治理

审批条件不应只留在审批评论里，应同步进入项目台账。

例如：

- 合同台账补录。
- 法务豁免确认。
- 供应商准入补充。
- 发票/回单上传。
- 项目编号修正。
- 预算科目确认。
- 解除协议补签。

## 六、建议指标体系

### A. 买种阶段指标

1. **潜在新业务入池数**  
   每周新增潜在新业务数量。

2. **潜在新业务准入卡完成率**  
   `已完成准入卡的潜在新业务数 / 潜在新业务总数`

3. **潜在新业务转正式项目率**  
   `转为正式项目数 / 潜在新业务总数`

4. **潜在新业务淘汰率**  
   `明确停止跟进的机会数 / 潜在新业务总数`

5. **技术路径明确率**  
   `已明确靶点、细胞类型、适应症和验证路径的机会数 / 潜在新业务总数`

6. **价值路径明确率**  
   `已明确 BD / License-out / 注册 / 收费治疗 / 数据资产路径的机会数 / 潜在新业务总数`

### B. 项目立项指标

1. **Gate 0 通过率**  
   `通过 Gate 0 的项目数 / 提交 Gate 0 评审项目数`

2. **项目可研报告完成率**  
   `已完成可研报告项目数 / 正式立项项目数`

3. **预算前置完成率**  
   `立项前完成预算测算项目数 / 正式立项项目数`

4. **周期计划完成率**

## 八、对前序措施和指标的修正

结合公司方向和项目任务分解，前序指标需要做三点升级：

1. **从审批质量升级为价值孵化质量**  
   审批指标仍重要，但要服务于买种、育苗、卖苗的价值链，而不是孤立追求审批效率。

2. **从项目进度升级为阶段证据资产**  
   不只看“是否推进”，还要看每个阶段是否沉淀科学证据包和市场证据包。

3. **从问题追踪升级为组织闭环**  
   条件通过、材料补充、风险事件、项目延误，都要进入 AI for Organisation 的规则复盘和流程迭代。

## 九、最终结论

结合公司方向、项目任务分解、周报和审批数据，当前问题可以概括为：

> 公司已经有“买种-育苗-卖苗”和 AI for Value 的顶层方向，也有项目全生命周期任务框架，但日常运营仍停留在机会驱动、事件驱动和审批补救驱动，尚未完全形成“阶段门控 + 并行任务流”、证据资产和经营指标闭环。

下一步最关键的不是再增加更多项目，而是把现有项目和潜在新业务纳入统一机制：

- 潜在新业务先入池。
- 正式项目原则上先过 Gate 0。
- 需要先行推进的任务必须标注依赖、风险和停止条件。
- 每个主阶段和并行任务流都有明确交付物。
- 每个项目沉淀科学证据包和市场证据包。
- 审批条件、预算、合同、供应商和风险事件进入运营治理看板。

只有这样，公司才能把医院意向和早期管线机会持续转化为可验证、可交易、可注册、可复盘的长期价值资产。
