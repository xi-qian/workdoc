NanoClaw Module Architecture

Host (Node.js) orchestrates per-session agent containers (Bun). Modules register hooks/handlers; container speaks to host only via two session DBs.

Host Process · Node.js

  • • Single Node process, pnpm workspace; entry at src/index.ts
  • • Core pipeline: Router → SessionManager → ContainerRunner → Delivery → HostSweep
  • • Startup: circuit breaker + upgrade tripwire + DB migrations + backfills
  • • Graceful shutdown via onShutdown registry (channels, polls, CLI server)
  • • Two delivery polls: active 1s (running sessions) + sweep 60s (all)

Cross-cutting Modules

  • • permissions: sender resolver · access gate · scope · interceptor + user-dm
  • • approvals: pickApprover → DM card · OneCLI bridge · response handler
  • • scheduling: pre-task scripts · recurrence fan-out (host + container)
  • • agent-to-agent: agent-route · create-agent · destinations table
  • • self-mod: install_packages / add_mcp_server with approval gate
  • • interactive · typing · mount-security: ask_question · setTyping · mount allowlist

Agent Container · Bun

  • • Per-agent-group Docker container, mounts session DBs read-only
  • • poll-loop: getPending → markProcessing → query → push (concurrent)
  • • Provider: claude (default) · mock · opencode/codex via skills
  • • MCP tools: core · agents · cli · interactive · scheduling · self-mod
  • • Container DB: bun:sqlite · DELETE journal · seq parity (odd)
  • • Skills mounted RO: onecli-gateway · welcome · self-customize · agent-browser · slack-formatting
